## 概述

抽屉浮窗是从屏幕边缘滑入的容器，适合需要吸引用户注意，但不希望完全打断当前任务的场景。

用户可点击面板外区域、执行返回操作，或点击关闭按钮退出。面板内应只承载核心信息和必要操作，避免内容层级过深或流程过于复杂。抽屉浮窗适用于手机、平板、折叠屏等多种设备形态。在 HyperOS 中，其布局会根据设备形态和屏幕尺寸自动调整，确保内容清晰、操作顺畅。

## 类型与结构

### 类型

![抽屉浮窗类型](/media/os4/components/containers/drawer/type.png)

| 类型 | 模态 | 背景关系 | 特性 |
|------|------|----------|------|
| 抽屉 | 模态类型 | 不可交互 | - 手机端的默认呈现形态<br />- 默认显示遮罩，打开后不可操作下层界面 |
| | 非模态类型 | 可交互 | - 手机端的默认呈现形态<br />- 抽屉任务与背景任务可并行处理，互不影响 |
| 浮窗 | 模态类型 | 不可交互 | - 折叠屏展开态和平板端的默认呈现形态<br />- 默认显示遮罩，打开后不可操作下层界面 |
| 近手浮窗 | 模态类型 | 不可交互 | - 以近手的形式展示在点击入口的旁边<br />- 不显示遮罩，但打开后仍不可操作下层界面 |

|------|------|------|

| 把手 | 可选 | - 用于提示可拖拽、切换高度或关闭 |

| 内容区 | 是 | - 展示核心内容、表单、列表、宫格、选择器或配置项 |

## 交互与状态

### 档位与停靠交互

![档位与停靠](/media/os4/components/containers/drawer/levels.png)

- 抽屉提供 S、M、L 三种档位，也支持业务自定义高度（[规格详见](https://mi.feishu.cn/wiki/DWUkw4EVEi74YQknaO6cBN36n5c#share-Kf9rd30lVo6j0px8AXOcP2snnsf)）
- 用户向上拖拽时，抽屉从 S 档进入 M 档或 L 档
- 继续向上到达 L 档后只回弹，不产生更高档位
- 用户向下拖拽时，抽屉从 L 档回到 M 档或 S 档
- 不存在多档位场景时，可以从 L 档下滑后跳过 M 档和 S 档，直接进入关闭 / 隐藏
- 内容超过 M 档容量时直接使用 L 档，不在 M-L 之间创建额外状态

### 关闭、返回与挽留

<Callout type="info">
  任务必须完成、内容未保存或操作不可中断时，应限制关闭。
</Callout>

| 关闭方式 | 模态抽屉 | 非模态抽屉 | 规则 |
|----------|----------|------------|------|
| 关闭/返回 | - 可提供关闭、取消或返回入口 | - 可关闭场景应提供明确入口 | - 返回顺序应可预期，不跳过未保存提示 |
| 点击遮罩 | - 默认等同关闭 | - 不适用，非模态抽屉不使用遮罩 | - 高价值任务关闭前应确认或提示保存 |
| 手势关闭 | - 下滑可关闭 | - 下滑可关闭 | |

### 拖拽、吸附与回弹

<Callout type="info">
  手指移动时，容器位置、遮罩透明度和背景关系应同步变化。
</Callout>

| 阶段 | 用户行为 | 系统行为 | 视觉与交互反馈 |
|------|----------|----------|----------------|
| 发起拖拽 | 按住抽屉把手、标题区或内容区并开始移动 | 识别触控位置、移动方向及抽屉当前档位 | 抽屉开始跟随手指移动 |
| 拖拽中 | 向上或向下拖动抽屉 | 根据位移、速度和方向，实时判断目标档位 | 抽屉高度随手指变化 |
| 释放拖拽 | 松开手指 | 将抽屉吸附至 S、M、L 或自定义档位 | 抽屉平滑过渡至目标高度 |
| 操作结束 | 抽屉到达目标位置 | — | 抽屉稳定停留 |

### 叉号与勾号

<Callout type="info">
  抽屉浮窗顶部的叉号和勾号应先定义语义，再决定位置。
</Callout>

![叉号与勾号语义](/media/os4/components/containers/drawer/close-icons.png)

| 控件 | 语义 | 位置 | 规则 |
|------|------|------|------|
| 叉号 / 取消 | - Cancel<br />- 放弃当前任务或退出正在编辑的流程 | 左侧 | - 左侧叉号应被理解为「取消」<br />- 如存在未保存内容或高风险放弃结果，应给出确认或保存提示 |
| 叉号 / 关闭 | - Close<br />- 结束信息查看或收起展示层 | 右侧 | - 右侧叉号应被理解为「关闭」<br />- 只承载信息展示、详情查看、说明预览等<br />- 关闭后不应丢失业务结果，也不应让用户误以为取消了某个提交 |
| 返回 | - Back<br />- 回到抽屉内上一层级或上一步 | 左侧 | - 返回只回退层级，不应等同于关闭整个抽屉<br />- 没有上一级时不展示返回 |
| 勾号 / 确认 | - Commit<br />- 确认当前选择、保存编辑或提交结果 | 右侧 | - 勾号只用于正向确认，不用于普通关闭<br />- 若任务结果重要，可使用清晰文案按钮 |
| 勾号 / 完成 | - Done<br />- 完成当前编辑或添加流程 | 右侧 | - 完成表示任务已结束<br />- 并回到发起上下文或上一层级 |
| 勾号 / 保存 | - Save<br />- 保存当前修改 | 右侧 | - 保存应只在有可保存结果时出现<br />- 保存后按业务返回上一层级或关闭抽屉 |
| 勾号 / 应用 | - Apply<br />- 应用当前设置但不一定关闭任务 | 右侧 | - 应用表示让当前配置生效<br />- 是否关闭抽屉由业务流程决定 |

#### 尺寸

![尺寸](/media/os4/components/containers/drawer/size.png)

| 档位 | Token |
|------|-------|
| S 档 | - 高度 = `抽屉内标题区+手柄的高度+底部操作杆的高度` |
| M 档 | - 高度 = `屏幕高度 * 70%` |
| L 档 | - 高度 = `屏幕高度 - 52dp` |
| 自适应高度 | - 内容在 S-M 之间时，根据内容多少自适应高度<br />- 最低不低于 70%；内容超过 M 时直接使用 L 档 |

#### 圆角

![圆角](/media/os4/components/containers/drawer/radius.png)

| 形态 | 规则 |
|------|------|
| 抽屉 | - `miuix_raidus_shape_2xl`<br />- `36dp` |
| 浮窗 | - `miuix_raidus_shape_2xl`<br />- `36dp` |

#### 布局

![布局](/media/os4/components/containers/drawer/layout.png)

| 元素 | 布局规则 |
|------|----------|
| 标题栏 | - 继承自标题栏组件<br />- 标题栏距离顶部 `6dp`<br />- 标题栏自身高度 `56dp` |
| 手柄 | - 自身尺寸 `56*3dp`<br />- 距离顶部 `6dp`<br />- 水平居中 |
| 内容区 | - 高度 = 抽屉自身高度 - `6dp` - `56dp` |

#### 热区

![热区](/media/os4/components/containers/drawer/hotspot.png)

- 把手、标题区或 ActionBar 可作为拖动热区
- 在绿色区域滑动时，优先响应抽屉的关闭或高度调整
- 在蓝色区域滑动时，优先响应内容滚动；当内容已滚动至顶部后，继续下滑可关闭抽屉或调整其高度

## 动效

![动效](/media/os4/components/containers/drawer/motion.png)

### 多重覆盖

<Callout type="info">
  当前抽屉需要打开新的抽屉时，新抽屉应作为独立的临时任务叠加在原抽屉之上。
</Callout>

| 要素 | 规则 |
|------|------|
| 标题栏 | - 新抽屉浮窗显示自身临时任务标题栏，不继承上一个抽屉浮窗标题 |
| 动效 | - 新抽屉浮窗从底部弹出并覆盖旧抽屉浮窗<br />- 旧抽屉浮窗可降低高度，并做轻微缩小后退 |
| 关闭抽屉浮窗 | - 按层级退出，先关闭新抽屉浮窗，再关闭旧抽屉浮窗 |
| 高度规格 | - 新抽屉浮窗尽可能使用相同高度或大一档位的高度，以完整覆盖旧抽屉浮窗 |

### 页面内切换

<Callout type="info">
  当任务仍属于同一个抽屉浮窗内的页面层级变化时，应保持容器不动，仅切换内部页面。
</Callout>

| 要素 | 规则 |
|------|------|
| 标题栏 | - 每级页面均应提供返回入口，通常使用左上角返回图标 |
| 动效 | - 抽屉容器保持不动<br />- 进入时页面从右向左切换，返回时从左向右切换 |
| 系统返回 | - 逐级返回上一页，直到没有内部页面后，再关闭抽屉浮窗 |

## 多端设计指南

### 多端适配规则

![多端适配](/media/os4/components/containers/drawer/multi-device.png)

<Callout type="info">
  抽屉浮窗会跟随设备和屏幕的变化进行自动的布局匹配，为用户提供最佳的使用体验。
</Callout>

| 策略 | 适用条件 |
|------|----------|
| 抽屉 → 浮窗 | - 在折叠机（展开态）和平板上的默认呈现形态 |
| 抽屉 → 抽屉 | - 当页面内元素数量不固定或内容较多时，为提升浏览效率，可使用抽屉<br />- 当用户需要与背景页面中的元素交互时，可使用非模态抽屉 |

### 窗口尺寸规则

![窗口尺寸](/media/os4/components/containers/drawer/window-size.png)

| 终端 | 类型 | 尺寸大小 | 规则 |
|------|------|----------|------|
| 手机 | 抽屉 | 宽度 = 贴合当前窗口<br />高度 = 按 S / M / L 规则计算 | W &lt; 747dp 或 H &lt; 628dp |
| Fold | 浮窗 | 宽度 = 546dp<br />高度 = 屏幕尺寸 × 80%（竖屏 Max 636dp） | W ≥ 747dp 且 H ≥ 628dp |
| Fold | 抽屉 | 宽度 = 628dp<br />高度 = 按 S / M / L 规则计算 | |
| Pad | 浮窗 | 宽度 = 546dp<br />高度 = 屏幕尺寸 × 80%（竖屏 Max 636dp） | |
| Pad | 抽屉 | 宽度 = 贴合当前窗口（Max 850dp）<br />高度 = 按 S / M / L 规则计算 | |
