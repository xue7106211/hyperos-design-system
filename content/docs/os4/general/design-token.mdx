## 文件位置

Design Tokens 以 **W3C DTCG** 风格 JSON 存放，按 **层 × 模式** 拆分：

```text
tokens/
  reference.light.json
  reference.dark.json
  semantic.light.json
  semantic.dark.json
  component.light.json
  component.dark.json
```

重新导入（Figma Variables 导出目录）：

```bash
npm run tokens:import -- /path/to/OS4Token
```

## 分层约定

| 层 | 前缀 | 用途 |
|----|------|------|
| **Reference** | `reference.*` | 原始色板（solid / opacity）与基础数值（space / radius） |
| **Semantic** | `semantic.*` | 语义色与间距圆角（bg / text / fill / padding / …） |
| **Component** | `component.*` | 组件级色值（navigation / input / toggles / …） |

Token 名保留 Figma / MIUIX 原名（一律 `miuix_*`，**不加**层或分组前缀）。`TokenTable` 的 `groups` 仅用于筛选，不出现在名称列。

示例 Token 名：

- `miuix_color_bg_layout_white_page`
- `miuix_comp_nav_color_fill_enabled`
- `miuix_blue_solid_level_500`

筛选示例：`groups={["semantic.bg"]}`、`groups={["component.navigation"]}`、`groups={["reference.solid.blue"]}`。

界面优先使用 **Semantic**，组件实现优先 **Component**；Reference 仅作底表，避免在业务里直接绑死。

## 主题模式

同一 Token 路径在 Light / Dark 下可能取不同值。下方 `TokenTable` 支持 **Light / Dark** 切换预览。

Typography（字体）Token 尚未纳入本仓库，待后续导出补充。

Figma Variables → tokens:import → tokens/*.{light,dark}.json → 文档站 TokenTable

## 使用原则

- 优先引用 **Semantic / Component**，而非 Reference 原始色阶
- 暗色模式通过同名 Token 的 Dark 值切换，避免业务硬编码 hex
- 确保文本与背景对比度满足 WCAG AA
